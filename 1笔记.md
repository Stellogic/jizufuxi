# IEEE 754

计算机中，小数不能直接顶点二进制标识，因为位数有限范围太小。参考科学计数法，这就是浮点数

1. 符号位s：1bit
   1. 定义：1位，最高位
   2. 0正1负
   3. 放高位可以用证书比较指令直接比较浮点数大小
2. 阶码E：8bit移码
   1. 位数：8位
   2. 表示方式：偏移码，偏移量为127
   3. 取值：一般1到254，0和255是特殊含义
   4. 定义：不是真实的指数值，而是真实指数+偏移量
   5. 因为真实指数可能是负数，偏移码把最小的指数映射到0
   6. 公式：E = 真实指数+127
   7. 1. $E=0$ 且 $M=0$ → 表示 ±0（零
      2. $E=0$ 且 $M \neq 0$ → 表示 非规格化数（隐含前导0，而不是前导1
      3. $E=255$ 且 $M=0$ → 表示 ±∞（无穷大）
      4. $E=255$ 且 $M \neq 0$ → 表示 NaN（非数值）
3. 尾数M：23bit隐含前导1。原码
   1. 尾数存储的时小数部分的二进制位，实际数值是1.+23位
   2. 位数：23位
   3. 实际尾数：1.M
   4. 例外：E为0时为0.M
4. 二进制小数转十进制
   1. 二进制每一位代表2的负次方
5. 数值公式： $\text{值} = (-1)^S \times 1.M \times 2^{E-127}$

# 编码方式

注意三种编码都有符号位

1. 原码：符号位（0正1负）+数值部分保持原样
   1. 但是0有两种看表示

2. 反码：正数同原码，负数=符号位1 + 对应正数按位取反
   1. 0有两种表示
3. 补码：正数同原码，负数 = 对应正数按位取反 + 1（反码+1）（符号位不参与）
   1. 只有一个0
   2. 但是范围不对称，-2^(n-1) 到 2^(n-1) - 1。因为 0 占用了一个"本该属于正数"的编码模式，所以正数少了一个，负数多了一个。

补码性质

1. 定义式：
   1. 若 X ≥ 0：([X]\_补 = X)（二进制原样表示）
   2. 若 X < 0：([X]\_补 = 2^n + X = 2^n - |X|)
2. 求补取负：由[x]的补码求[-x]的补码
   1. 连同符号位一起，各位取反，末位+1
3. 求真值：最高位是负权重，其他都是正

# 机器数

1. 将符号数字化的二进制编码
   1. 符号位（0正1负）+数值位
   2. 真值：实际数值
2. 常见有原码，反码，补码，移码。
   1. 移码是补码符号位取反，用于浮点数的阶码
   2. 移码 = 真值 + 偏移量。移码此时是无符号数
   3. 补码可以消除减法：
      1. A-B = A + (-B)
      2. -B的补码就是B的 补码连带符号位取反后+1
