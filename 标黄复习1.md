# 性能指标：

1. 主频：cpu时钟频率。一秒几个周期
2. 时钟周期：主频的倒数。
3. cpi：cycles per instruction：每条指令平均需要几个时钟周期
4. cpu执行时间：指令条数 * cpi * 时钟周期
5. MIPS：每秒执行几百万条指令。指令条数/（执行时间*10^6）或者主频/(cpi*10^6)
6. 吞吐率：单位时间完成多少任务
7. 响应时间：完成一个任务要多久

# 硬件组成

| 部件   | 作用          |
| ---- | ----------- |
| 运算器  | 做算术运算、逻辑运算  |
| 控制器  | 取指、译码、发控制信号 |
| 存储器  | 存程序和数据      |
| 输入设备 | 把外部信息送入计算机  |
| 输出设备 | 把计算结果输出     |

**CPU=运算器+控制器+寄存器组**

CPU 通过**总线**和主存、I/O 设备连接。总线通常分为**地址总线、数据总线、控制总线**。地址总线负责指出“访问哪里”，数据总线负责传数据，控制总线负责读/写、时钟、中断等控制信号。总线资料一般也这样划分三类信号线。

## 原码反码补码移码

原码：最高位是符号位，0 表示正，1 表示负，其余位表示绝对值。0有两个

反码：正数反码等于原码；负数反码是“符号位不变，数值位取反”。

补码：正数补码等于原码；负数补码是“反码 + 1”。

第一，只有一个 0。

第二，加减法可以统一成加法。

第三，最高位既是符号位，又参与运算。

移码：常用于浮点数阶码。

移码=真值+2^(n−1)

## 校验码

奇偶校验码是在原数据后面加 1 位校验位，让整个编码中 1 的个数满足某种规则。

偶校验：让 1 的总个数为偶数。

奇校验：让 1 的总个数为奇数。

奇偶校验能检测奇数个比特错误，尤其是单比特错误；但如果同时错了两个 bit，它可能检测不出来。资料中也通常强调 parity bit 是附加位，用来让 1 的个数满足奇/偶规则，主要用于错误检测。

## cache：

### 局部性

时间局部性：刚访问过的数据，短期内可能再次访问。

直接映射：Cache行号=主存块号modCache行数

### 替换策略

| 策略     | 含义              |
| ------ | --------------- |
| FIFO   | 替换最早进入 Cache 的块 |
| LRU    | 替换最近最少使用的块      |
| LFU    | 替换访问次数最少的块      |
| Random | 随机替换            |


### 写

#### 命中

写直达 Write Through：同时写 Cache 和主存。

优点是主存数据总是最新；缺点是写操作慢。

写回 Write Back：只写 Cache，等块被替换时再写回主存。

优点是减少主存写次数；缺点是控制复杂，需要脏位 Dirty Bit。

#### 未命中

写分配 Write Allocate：先把块调入 Cache，再写。

非写分配 No Write Allocate：直接写主存，不调入 Cache。

### 虚拟存储器：让程序以为自己拥有一个很大的连续地址空间，但实际上数据可能在主存，也可能在磁盘。


与cache相同

| 相同点      | 说明                       |
| -------- | ------------------------ |
| 都利用局部性原理 | 常用的内容放在更快层次              |
| 都有映射     | 主存块到 Cache；虚页到物理页        |
| 都有命中/失效  | Cache miss；缺页 page fault |
| 都有替换策略   | 替换 Cache 块；替换页面          |


| 对比   | Cache      | 虚拟存储器       |
| ---- | ---------- | ----------- |
| 解决矛盾 | CPU 与主存速度差 | 主存容量不足      |
| 数据单位 | 块/行        | 页/段         |
| 管理者  | 主要由硬件管理    | OS + 硬件共同管理 |
| 对程序员 | 基本透明       | 提供虚拟地址空间    |
| 失效代价 | 访问主存       | 访问磁盘，代价极大   |

#### 页式

地址分为：虚页号+页内偏移

用**页表**把虚页号转换成物理页框号：

物理地址=页框号+页内偏移

优点：页大小固定，方便管理，没有外部碎片。

缺点：可能有内部碎片，页表可能很大。

#### 段

地址：段号+段内偏移

段表中保存段基址和段长。

物理地址=段基址+段内偏移

优点：符合程序逻辑，比如代码段、数据段、栈段，便于保护和共享。

缺点：段长不固定，容易产生外部碎片。

#### 段页式虚拟存储器

先分段，段内再分页。

地址：段号+页号+页内偏移

先查段表，找到该段对应的页表；

再查页表，找到页框号；

最后拼接页内偏移，形成物理地址。

优点：兼有段式的逻辑结构和页式的固定大小管理。

缺点：地址转换更复杂，访问表的次数更多。

# 指令系统和寻址

## 指令系统

### 指令：让cpu执行某个操作的二进制命令：操作码+地址码

操作码说明“做什么”，地址码说明“对谁做”。

### 指令系统：一台机器支持的全部指令集合。

### 指令格式：指令内部各字段如何安排。

| 字段      | 含义         |
| ------- | ---------- |
| 操作码 OP  | 指明操作类型     |
| 源操作数地址  | 数据从哪里来     |
| 目的操作数地址 | 结果放哪里      |
| 寻址方式字段  | 说明如何找操作数   |
| 立即数/位移量 | 直接给出的常数或偏移 |

## 指令、数据寻址方式

寻址方式的本质就是：操作数在哪里，怎么找到它。

| 寻址方式    | 有效地址 EA   | 操作数怎么拿           |
| ------- | --------- | ---------------- |
| 立即寻址    | 不需要 EA    | 操作数就在指令里         |
| 直接寻址    | EA=A      | 到内存地址 A 取数       |
| 间接寻址    | EA=(A)    | 先到 A 取地址，再用该地址取数 |
| 寄存器寻址   | 不访问内存     | 操作数在寄存器里         |
| 寄存器间接寻址 | EA=(R)    | 寄存器里放的是地址        |
| 变址寻址    | EA=A+(IX) | 地址字段加变址寄存器       |
| 基址寻址    | EA=A+(BR) | 地址字段加基址寄存器       |
| 相对寻址    | EA=(PC)+A | PC 加偏移量          |
| 堆栈寻址    | 栈顶        | 操作数默认在栈顶         |

立即寻址：指令里直接是数。
例如 MOV R1, #5，5 就是操作数。

直接寻址：指令里是地址。
例如 MOV R1, [1000H]，要去内存 1000H 取数。

间接寻址：指令里给的是“地址的地址”。

## 2. 微程序控制器的基本思想

CPU 执行机器指令时，其实要拆成很多更小的控制动作。

这些小动作叫微操作。

控制这些微操作的信号叫微命令。

微程序控制器的思想是：

**把控制信号做成一条条微指令，存放在控制存储器中。执行一条机器指令时，就去控制存储器中取出一段微程序，逐条发出控制信号**。

**机器指令→微程序→微指令→微命令→微操作**

| 名称  | 含义                 | 例子                 |
| --- | ------------------ | ------------------ |
| 微操作 | 最基本的数据传送或运算动作      | PC→MAR             |
| 微命令 | 控制某个微操作的控制信号       | 允许 PC 输出、允许 MAR 接收 |
| 微指令 | 一组微命令的集合           | 同时控制多个部件动作         |
| 微程序 | 多条微指令组成，用来实现一条机器指令 | ADD 指令的微程序         |

**微命令控制微操作，微指令包含微命令，微程序包含微指令，机器指令由微程序实现。**

# I/O、中断、DMA、通道

## cpu与外围方式

| 方式           | 特点                  |
| ------------ | ------------------- |
| 程序直接控制/无条件传送 | CPU 直接读写设备，不检查状态    |
| 程序查询方式       | CPU 不断查询设备是否准备好     |
| 中断方式         | 设备准备好后主动通知 CPU      |
| DMA 方式       | DMA 控制器直接在内存和设备间传数据 |
| 通道方式         | 专门 I/O 处理器执行 I/O 程序 |

程序查询方式简单，但 CPU 会浪费大量时间等待设备。

中断方式比查询方式好，因为 CPU 不必一直等，可以先执行别的程序。

DMA 更适合大量数据块传输，比如磁盘和内存之间搬运数据。DMA 的核心是让 DMA 控制器接管总线，在设备和内存之间直接传输数据，传完后再通知 CPU。

## 外围编址

| 编址方式 | 特点                 |
| ---- | ------------------ |
| 统一编址 | I/O 端口和内存使用同一个地址空间 |
| 独立编址 | I/O 端口有单独地址空间      |


统一编址也叫存储器映射 I/O。
优点是可以用普通访存指令访问设备寄存器。
缺点是占用一部分内存地址空间。

独立编址也叫 I/O 映射。
优点是不占用内存地址空间。
缺点是需要专门 I/O 指令，比如 IN、OUT。

1. CPU 向设备发启动命令。
2. CPU 读取设备状态寄存器。
3. 判断 Ready 位是否为 1。
4. 如果没准备好，继续查询。
5. 如果准备好，进行数据传送。
6. 判断是否传送完毕。
7. 没完继续查询，完成后结束。

## 中断：CPU 正在执行程序时，外部设备或内部事件请求 CPU 暂停当前程序，转去执行中断服务程序，处理完后再回到原程序。

| 概念         | 含义              |
| ---------- | --------------- |
| 中断源        | 发出中断请求的来源       |
| 中断请求       | 设备向 CPU 发出的请求信号 |
| 中断响应       | CPU 决定接受中断      |
| 中断服务程序 ISR | 处理中断的程序         |
| 中断向量       | 中断服务程序入口地址      |
| 中断优先级      | 多个中断同时来时先处理哪个   |
| 中断屏蔽       | 暂时禁止某些中断        |
| 中断嵌套       | 高优先级中断打断低优先级中断  |

**中断请求→中断判优→中断响应→保存现场→执行中断服务程序→恢复现场→返回原程序**

和子程序

| 对比   | 中断         | 子程序调用    |
| ---- | ---------- | -------- |
| 发生原因 | 随机、外部或异常事件 | 程序主动调用   |
| 时间   | 不可预测       | 程序安排好的   |
| 返回地址 | 当前被打断的位置   | 调用指令的下一条 |
| 处理内容 | I/O、异常、时钟等 | 普通功能模块   |

## 通道

通道可以理解成专门负责 I/O 的处理器。
