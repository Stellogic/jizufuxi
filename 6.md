【判断题】
采用写回法（Write Back）策略时，cache无需使用写缓冲。

**cache写操作策略**

写比读复杂。读：CPU 要读数据 → 先查 cache → 命中就读 cache → 不命中再访问主存。

但是写的话，因为系统中至少两份数据，cache和主存中的。cache改了，主存是否要改？

**写直达法（write through）**：写cache同时写主存。cache和主存始终一致。但是主存速度比cache慢。因此引入 **写缓冲**

写缓冲：先把要写入主存的数据放进写缓冲，让 CPU 继续执行，主存慢慢写。

**写回法（write back）**：CPU 写数据时，只修改 cache，不立即修改主存。只有当该 cache 块被替换出去（cache空间不够了，要装入新的主存块时，要腾出一个原来的cache块）时，才写回主存。但是可能会有cache和主存不一致，为了记录cache被修改过，**要用脏位**

脏位：cache的一个标志位，表示这个cache是否修改过，是否需要写回。0代表未被修改，1代表已经修改。

但是当脏块被替换时，依然要写回。写回法不需要每次写都使用写缓冲，但在脏块替换写回主存时，仍然可以使用写缓冲。

【判断题】
增大cache块大小可能提高空间局部性，但可能增加失效损失。

**空间局部性**：程序访问了某个存储单元后，接下来很可能访问它附近的存储单元。Cache 不会只把你当前访问的一个字节或一个整数拿进来，而是会把它附近的一整块数据一起拿进来。这样如果程序接下来访问附近数据，就能直接命中 Cache。

**cache块大小** 指一次从主存调入 Cache 的数据块大小。

**Cache miss**，中文常叫 Cache 未命中 或 缓存失效。

**失效损失** 指 Cache miss 后从下一级存储器取回数据所花的额外时间。

【判断题】
全写法策略在写失效时将主存数据块调入cache，通常与写回法配合使用。

**全写法就是写直达法**

| 场景 | 常见策略 | 含义 |
|---|---|---|
| 写命中 | 写直达 / 全写法 | 同时写 Cache 和主存 |
| 写命中 | 写回法 | 只写 Cache，块被替换时再写回主存 |
| 写失效 | 写分配法 | 先把主存块调入 Cache，再写 Cache |
| 写失效 | 非写分配法 | 不调入 Cache，直接写主存 |

写分配法和非写分配法都是处理cache写失效的策略。就是cpu要写某个地址，但是数据库不在cache中怎么办

1. 写分配：写失效时，先把主存中的这个数据块调入 Cache，然后再在 Cache 中完成写操作。思想是既然程序现在要写这个地址，那么它之后很可能还会继续访问附近的数据。把这个块放进 Cache，后续访问可能更快。局部性原理，尤其是时间局部性和空间局部性。
2. 非写分配法：写失效时，不把主存块调入 Cache，而是直接把数据写到主存中。

全相联映射的优点是冲突率低，但缺点是硬件成本高。

**1. 直接映射**：主存中的每一块，只能放到 Cache 中某一个固定位置。例如cache8行，Cache 行号 = 主存块号 mod Cache 行数。优点时快简单成本低，但是冲突率高。比如 Cache 第 0 行已经放了主存块 0，现在又要访问主存块 8，虽然 Cache 其他行可能是空的，但 8 只能放第 0 行，所以必须把 0 挤掉。这就叫冲突。
