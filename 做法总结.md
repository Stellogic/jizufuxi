# 一、简答题直接背

## 1. 冯·诺依曼计算机思想

冯·诺依曼计算机的核心思想是**存储程序、程序控制**。

要点：

1. 数据和指令均用二进制表示。
2. 指令由操作码和地址码组成。
3. 程序和数据都存放在存储器中。
4. CPU按顺序取指执行，分支由转移指令实现。
5. 计算机由**运算器、控制器、存储器、输入设备、输出设备**五大部件组成。

## 2. 指令和数据如何区分

背两点：

**时间上**：取指周期从内存读出的是指令；执行周期读出或写入的是数据。

**空间上**：指令流向控制器；数据在存储器、寄存器、运算器之间流动。

## 3. 计算机性能指标

必须背：

* **机器字长**：CPU一次能处理的二进制位数。
* **主频**：CPU时钟频率。
* **时钟周期**：主频的倒数。
* **CPI**：每条指令平均需要的时钟周期数。
* **CPU执行时间** = 指令条数 × CPI × 时钟周期。
* **MIPS** = 指令条数 / 执行时间 / 10⁶ = 主频 / CPI / 10⁶。

## 4. 存储器层次结构

存储器按速度、容量、价格形成层次：

`寄存器 → Cache → 主存 → 辅存`

越靠近CPU，速度越快、容量越小、价格越高。

主存主要指标：

* 存储容量
* 存取时间
* 存取周期
* 存储器带宽

## 5. SRAM、DRAM、ROM

**SRAM**：速度快，不需要刷新，常用于Cache，集成度低，成本高。

**DRAM**：容量大、成本低，需要刷新，常用于主存。

**ROM**：只读或主要用于读，掉电后信息不丢失。

DRAM刷新方式：

1. 集中刷新
2. 分散刷新
3. 异步刷新

## 6. Cache基本思想

Cache利用程序的**局部性原理**提高访存速度。

局部性包括：

* **时间局部性**：刚访问过的数据可能很快再次访问。
* **空间局部性**：刚访问地址附近的数据可能很快被访问。

Cache与主存之间以**块**为单位交换数据。

## 7. Cache和虚拟存储器区别

**Cache**解决CPU和主存速度不匹配问题，由硬件自动完成，用户不可见。

**虚拟存储器**解决主存容量不够问题，把主存和辅存统一管理，由操作系统和硬件共同完成，用户感觉有更大存储空间。

Cache缺失只访问主存，代价较小；缺页要访问磁盘，代价很大。

## 8. 页式、段式、段页式虚拟存储

**页式**：按固定大小分页，页内地址不变，管理简单，无外部碎片，但有内部碎片。

**段式**：按程序逻辑分段，便于共享和保护，但会产生外部碎片。

**段页式**：先分段、段内分页，结合二者优点，但地址转换复杂。

地址转换核心：

`虚拟地址 = 虚页号 + 页内地址`

`物理地址 = 物理页号 + 页内地址`

页内地址不变，只替换页号。

## 9. 指令系统与指令格式

指令一般由：

`操作码 OP + 地址码 A`

组成。

操作码说明“做什么”，地址码说明“操作数在哪里”。

常见指令类型：

* 零地址指令
* 一地址指令
* 二地址指令
* 三地址指令

指令字长可以是定长，也可以是变长。

## 10. 寻址方式

必须背有效地址 EA：

| 寻址方式    | 含义            |
| ------- | ------------- |
| 立即寻址    | 操作数在指令中       |
| 直接寻址    | EA = A        |
| 间接寻址    | EA = (A)      |
| 寄存器寻址   | 操作数在寄存器中      |
| 寄存器间接寻址 | EA = (R)      |
| 变址寻址    | EA = A + (IX) |
| 基址寻址    | EA = A + (BR) |
| 相对寻址    | EA = A + (PC) |

看到“地址在寄存器中”就是寄存器间接；看到“寄存器内容 + 位移量”可能是变址、基址、相对。

## 11. CISC与RISC

**CISC**：指令多、格式复杂、寻址方式多、执行时间不固定，控制复杂。

**RISC**：指令少、格式规整、寻址方式少，多数指令一个周期完成，适合流水线。

## 12. CPU组成与指令周期

CPU由：

* 运算器
* 控制器
* 寄存器组
* 内部总线

组成。

指令周期包括：

`取指 → 间址 → 执行 → 中断`

并不是每条指令都有间址周期和中断周期，但一定有取指周期和执行周期。

## 13. 微程序控制器

背概念：

* **微命令**：最基本的控制信号。
* **微操作**：由微命令控制完成的基本操作。
* **微指令**：一组微命令 + 下一条微地址信息。
* **微程序**：实现一条机器指令的一段微指令序列。
* **控制存储器 CM**：存放微程序。

机器指令和微程序关系：

一条机器指令由一段微程序解释执行；一条微程序由若干微指令组成；一条微指令发出若干微命令。

## 14. 流水线

流水线思想：把一条指令分成多个阶段，多条指令重叠执行。

常见五段：

`IF 取指 → ID 译码 → EX 执行 → MEM 访存 → WB 写回`

流水线问题：

1. 结构相关：资源冲突。
2. 数据相关：前后指令数据依赖，最常见 RAW 写后读。
3. 控制相关：分支跳转导致取错指令。

## 15. 总线

总线是多个部件共享的信息传输通路。

按传送信息分：

* 数据总线
* 地址总线
* 控制总线

总线仲裁两类：

* 集中仲裁
* 分布仲裁

总线定时：

* 同步定时
* 异步定时
* 半同步定时

## 16. I/O方式

必须背：

| I/O方式 | 特点                      |
| ----- | ----------------------- |
| 程序查询  | CPU反复查询外设状态，硬件简单，但浪费CPU |
| 程序中断  | 外设主动请求CPU服务，适合随机事件      |
| DMA   | 外设和主存直接传送，适合高速、大批量数据    |
| 通道    | 通道统一管理外设，进一步解放CPU       |
| 外围处理机 | 更独立，接近一般处理机             |

DMA和程序中断区别：

DMA中断只是通知CPU一批数据传送结束；程序中断是CPU转去执行中断服务程序完成数据传送。

CPU响应中断条件：

1. CPU开中断。
2. I/O设备提出请求且未被屏蔽。
3. 经中断排队后被选中。
4. 响应时间一般在一条指令执行结束后。

# 二、计算题直接背模板

## 1. 原码、反码、补码、移码

正数：

`原码 = 反码 = 补码`

负数：

* 原码：符号位1，数值位写绝对值。
* 反码：符号位不变，数值位取反。
* 补码：反码 + 1。
* 移码：补码符号位取反。

n位补码整数范围：

`-2^(n-1) ~ 2^(n-1)-1`

例如8位补码范围：

`-128 ~ +127`

## 2. 补码加减法

做题步骤：

1. 写出两个数的补码。
2. 减法变加法：`x - y = x + (-y)`。
3. 符号位一起参与运算。
4. 结果仍是补码。
5. 判断是否溢出。

溢出判断：

同号相加，结果异号，溢出。

正 + 正 = 负：正溢出。
负 + 负 = 正：负溢出。

也可以用：

最高数值位进位和符号位进位不同，则溢出。

## 3. IEEE754单精度

格式：

`1位符号位 S + 8位阶码 E + 23位尾数 M`

公式：

`真值 = (-1)^S × 1.M × 2^(E-127)`

十进制转IEEE754步骤：

1. 转二进制。
2. 规格化成 `1.xxx × 2^e`。
3. 符号位 S：正数0，负数1。
4. 阶码 E = e + 127。
5. 尾数 M 取小数点后23位。
6. 拼成32位，再转十六进制。

IEEE754反算步骤：

拆 S、E、M → e = E - 127 → 写成 `±1.M × 2^e` → 转十进制。

## 4. 浮点加减法

步骤固定背：

1. 对阶：小阶向大阶看齐，尾数右移。
2. 尾数加减。
3. 规格化。
4. 舍入。
5. 判断溢出。

口诀：**对阶、运算、规格化、舍入、判溢出**。

## 5. 存储器容量扩展

核心公式：

`芯片数 = 目标容量 / 单片容量`

容量统一成：

`字数 × 位数`

先算**位扩展**，再算**字扩展**。

位扩展：扩字长。
例如 8位芯片组成32位，需要4片并联。

字扩展：扩字数。
例如 512K扩成1M，需要2组，用高位地址线片选。

地址线：

容量为 `2^n` 个单元，需要 n 根地址线。

片选线：

有 `2^k` 组芯片，需要 k 根高位地址线接译码器。

典型答法：低位地址线接所有芯片地址端，高位地址线经过译码器产生片选信号，数据线按位接CPU数据总线。习题中就用高位地址线通过译码器选择不同内存条。

## 6. Cache命中率和平均访问时间

命中率：

`h = Cache命中次数 / 总访问次数`

平均访问时间：

`Ta = h × Tc + (1-h) × Tm`

其中：

* Tc：Cache访问时间
* Tm：主存访问时间

效率：

`e = Tc / Ta`

## 7. Cache地址映射

直接映射地址格式：

`标记 Tag | Cache行号 | 块内地址`

组相联地址格式：

`标记 Tag | 组号 | 块内地址`

字段位数：

* 块内地址位数 = `log₂(块大小)`
* Cache行号位数 = `log₂(Cache行数)`
* 组号位数 = `log₂(组数)`
* 标记位数 = 主存地址位数 − 组号/行号位数 − 块内地址位数

直接映射：

`Cache行号 = 主存块号 mod Cache行数`

组相联：

`Cache组号 = 主存块号 mod Cache组数`

习题里直接映射就是先算主存块数、Cache行数，再分出标记、行号、块内地址。

## 8. 虚拟地址转换

页式虚存：

`虚拟地址 = 虚页号 + 页内地址`

`物理地址 = 物理页号 + 页内地址`

做题步骤：

1. 页面大小 = `2^p`，所以页内地址 p 位。
2. 虚拟地址高位是虚页号。
3. 查页表，虚页号换成物理页号。
4. 页内地址不变。
5. 拼接得到物理地址。

例题中页面大小1KB，所以页内地址低10位；高位是虚页号，查到对应物理页号后拼接得到物理地址。

## 9. FIFO和LRU页面置换

FIFO：淘汰最早进入主存的页。

LRU：淘汰最长时间未被访问的页。

做题方法：

画表，按访问序列一个个放页框：

* 命中：不缺页。
* 未命中且有空页框：装入，缺页。
* 未命中且无空页框：按FIFO或LRU淘汰，缺页。

## 10. 指令格式字段计算

操作码位数：

`OP位数 = ceil(log₂ 指令条数)`

寄存器字段位数：

`寄存器位数 = ceil(log₂ 寄存器个数)`

寻址方式字段位数：

`寻址方式位数 = ceil(log₂ 寻址方式种数)`

剩余位数给地址码。

变长编码题：分别算每类指令长度，再按出现比例求平均长度，和定长指令比较。

## 11. 流水线计算

流水线周期：

`T = max(各段时间)`

k段流水线执行n条指令：

`总时间 = (k + n - 1)T`

吞吐率：

`TP = n / 总时间`

加速比：

`S = 非流水时间 / 流水时间`

如果每段时间相等：

`S = nk / (k+n-1)`

画时空图：横轴时间，纵轴阶段，每条指令斜着排。习题中五段流水线按 IF、ID、EX、M、WB 排布。

## 12. 微指令字段设计

控制存储器容量为 `2^n` 个单元，则后继微地址字段需要 n 位。

微指令总位数固定时：

`微命令字段 = 微指令总位数 - 后继地址字段 - 判别/测试字段`

例如控存512个单元，因为 `512 = 2^9`，所以后继微地址字段9位。

## 13. 总线带宽

公式：

`总线带宽 = 总线宽度 × 总线频率`

注意单位换算：

8 bit = 1 Byte。

如果一次传送32位，总线周期50ns：

每秒传送次数 = `1 / 50ns`

带宽 = 32位 × 每秒传送次数。

## 14. 磁盘计算

磁盘容量：

`容量 = 记录面数 × 每面磁道数 × 每道扇区数 × 每扇区字节数`

平均旋转等待时间：

`平均等待时间 = 转一圈时间 / 2`

数据传输率：

`传输率 = 每道容量 × 每秒转数`

## 15. I/O CPU占用率

公式：

`CPU占用率 = 每秒I/O次数 × 每次服务所需时钟周期 / CPU每秒时钟周期数`

程序查询通常CPU占用高；中断较低；DMA最低。

# 三、应用题直接套模板

## 1. 存储器组成框图题

答题模板：

先算总芯片数：

`芯片数 = 目标容量 / 单片容量`

然后写：

1. 若单片位数不足，先进行位扩展。
2. 若单片字数不足，再进行字扩展。
3. 低位地址线接所有芯片地址端。
4. 高位地址线接译码器，译码器输出片选信号。
5. 数据线按位接CPU数据总线。
6. 读写控制线接所有芯片。

画图不用太精美，重点是：**地址线、数据线、片选线、读写线**都要体现。

## 2. 指令周期流程题

取指周期通用微操作：

`PC → MAR`

`M[MAR] → MDR`

`MDR → IR`

`PC + 指令长度 → PC`

`IR译码`

访存取数：

`有效地址 → MAR`

`M[MAR] → MDR`

`MDR → 寄存器`

存数：

`有效地址 → MAR`

`寄存器 → MDR`

`MDR → M[MAR]`

## 3. 寻址方式应用题

看到题目先判断寻址方式，再套EA公式：

* 指令里就是数：立即寻址。
* 指令里给地址：直接寻址。
* 指令里地址的内容才是地址：间接寻址。
* 操作数在寄存器：寄存器寻址。
* 地址在寄存器：寄存器间接。
* 寄存器 + 位移量：变址/基址/相对。

最后求操作数：

`先求EA，再到EA对应内存取操作数`

## 4. 微程序题

答题模板：

1. 机器指令由微程序实现。
2. 微程序由微指令组成。
3. 微指令包含操作控制字段和顺序控制字段。
4. 操作控制字段产生微命令。
5. 顺序控制字段决定下一条微指令地址。
6. 控存容量决定微地址字段位数。

## 5. 流水线应用题

答题模板：

1. 先列阶段：IF、ID、EX、MEM、WB。
2. 无冲突时，每周期进入一条新指令。
3. 遇到数据相关，后续指令停顿。
4. 遇到访存资源冲突，IF或MEM停顿。
5. 遇到分支，可能发生控制相关。
6. 计算总周期，看最后一条指令WB结束时间。
