# 硬布线控制器讲解与题型总结

## 0. 一句话先背

**硬布线控制器**就是用组合逻辑电路直接产生控制信号的控制器。

它根据：

- 指令操作码
- 指令周期/机器周期
- 节拍信号
- 状态条件信号

直接生成各个微操作所需的控制信号。

背诵版：

> 硬布线控制器由指令译码器、节拍发生器、状态条件逻辑和组合逻辑电路组成。它根据操作码、时序信号和状态标志直接产生控制信号，控制 CPU 各部件完成取指、译码和执行。其速度快，但结构复杂，修改和扩展困难。

---

## 1. 控制器到底在干什么

CPU 执行一条指令，本质上就是一串**微操作**按顺序发生。

例如取指令：

```text
PC -> MAR
M[MAR] -> MDR
MDR -> IR
PC + 1 -> PC
```

这些动作不会自己发生，必须由控制器发出控制信号：

- `PCout`：让 PC 的内容送到总线
- `MARin`：让 MAR 接收总线内容
- `MemRead`：读主存
- `MDRin`：让 MDR 接收主存数据
- `MDRout`：让 MDR 送到总线
- `IRin`：让 IR 接收指令
- `PC+1`：PC 自增

所以控制器的作用就是：

> 在正确的时间，给正确的部件发正确的控制信号。

---

## 2. 硬布线控制器组成

硬布线控制器常见组成：

| 组成部分 | 作用 |
| --- | --- |
| 指令寄存器 IR | 保存当前指令 |
| 指令译码器 | 根据操作码判断是哪条指令 |
| 时序发生器 | 产生机器周期、节拍脉冲，如 `T0、T1、T2` |
| 状态条件电路 | 提供条件信号，如 `Z、N、C、V、中断请求` |
| 组合逻辑电路 | 综合操作码、节拍、状态，产生控制信号 |

核心关系：

```text
控制信号 = f(操作码, 节拍信号, 状态条件)
```

考试写法：

> 硬布线控制器以组合逻辑电路为核心，输入包括指令译码信号、时序节拍信号和状态条件信号，输出为各类微操作控制信号。

---

## 3. 硬布线 vs 微程序

这是最容易考简答和选择的对比。

| 对比项 | 硬布线控制器 | 微程序控制器 |
| --- | --- | --- |
| 控制信号来源 | 组合逻辑电路直接产生 | 控制存储器中的微指令产生 |
| 速度 | 快 | 较慢 |
| 结构 | 复杂 | 较规整 |
| 修改扩展 | 困难，需要改硬件 | 容易，改微程序 |
| 适合 | RISC、指令简单、速度要求高 | CISC、指令复杂、便于扩展 |

背诵模板：

> 硬布线控制器速度快，因为控制信号由组合逻辑直接产生，不需要逐条读取微指令；但控制逻辑复杂，设计和修改困难。微程序控制器把控制信号编码在微指令中，结构清晰，便于修改和扩展，但需要访问控制存储器，速度相对较慢。

选择题关键词：

- 看到**速度快、RISC、组合逻辑、修改困难**：选硬布线。
- 看到**控制存储器、微指令、微程序、便于修改、CISC**：选微程序。

---

## 4. 硬布线控制信号怎么产生

假设某个控制信号 `C` 在以下情况为 1：

- 取指周期 `T0` 需要它
- `ADD` 指令执行阶段 `T3` 需要它
- `LOAD` 指令执行阶段 `T4` 也需要它

那么控制信号逻辑式就是：

```text
C = T0 + ADD·T3 + LOAD·T4
```

这里：

- `+` 表示或 OR
- `·` 表示与 AND
- `ADD`、`LOAD` 是指令译码器输出
- `T0、T3、T4` 是节拍信号

通用套路：

```text
某控制信号 = 所有需要该控制信号的微操作条件相或
```

也就是：

```text
C = 条件1 + 条件2 + 条件3 + ...
```

每个条件一般长这样：

```text
指令译码信号 · 周期/节拍信号 · 状态条件
```

## 5. 题型一：简答硬布线控制器

### 问法

- 什么是硬布线控制器？
- 硬布线控制器由哪些部分组成？
- 硬布线控制器有什么优缺点？

### 答题模板

硬布线控制器是用组合逻辑电路直接产生控制信号的控制器。它通常由指令译码器、时序发生器、状态条件逻辑和组合逻辑电路组成。控制器根据指令操作码、机器周期或节拍信号以及状态标志，产生各部件所需的控制信号，从而完成取指、译码、执行等操作。

优点是速度快，适合指令系统较简单、强调执行效率的机器；缺点是控制逻辑复杂，设计、调试、修改和扩展困难。

---

## 6. 题型二：硬布线与微程序比较

### 问法

- 比较硬布线控制器和微程序控制器。
- 为什么 RISC 多采用硬布线控制？
- 为什么 CISC 多采用微程序控制？

### 答题模板

硬布线控制器通过组合逻辑电路直接产生控制信号，执行速度快，但电路复杂，修改困难；微程序控制器把控制信号写入微指令并存放在控制存储器中，执行时逐条读出微指令产生控制信号，结构规整，便于修改扩展，但速度较慢。

RISC 指令数量少、格式规整、执行过程简单，适合用硬布线控制提高速度；CISC 指令复杂、寻址方式多、控制流程复杂，适合用微程序控制降低设计难度。

## 7. 题型三：根据微操作写控制信号逻辑式

### 做法步骤

1. 找出题目给了哪些指令，例如 `ADD、SUB、LOAD、STORE、JMP`。
2. 找出每条指令在每个节拍执行哪些微操作。
3. 盯住题目要求的控制信号，比如 `PCout、MARin、Read、IRin`。
4. 哪些节拍用到了这个控制信号，就把对应条件列出来。
5. 用或运算把所有条件相加。

### 示例

题目给出：

```text
T0: PC -> MAR
T1: M[MAR] -> MDR, PC + 1 -> PC
T2: MDR -> IR
ADD 指令 T3: R1 -> A
ADD 指令 T4: R2 -> B
ADD 指令 T5: A + B -> R1
LOAD 指令 T3: IR(addr) -> MAR
LOAD 指令 T4: M[MAR] -> MDR
LOAD 指令 T5: MDR -> R1
```

问 `MARin` 的逻辑式。

分析：

- `T0: PC -> MAR` 需要 `MARin`
- `LOAD 指令 T3: IR(addr) -> MAR` 需要 `MARin`

所以：

```text
MARin = T0 + LOAD·T3
```

问 `MemRead` 的逻辑式。

分析：

- `T1: M[MAR] -> MDR` 是取指读存储器
- `LOAD 指令 T4: M[MAR] -> MDR` 是执行阶段读存储器

所以：

```text
MemRead = T1 + LOAD·T4
```

问 `R1in` 的逻辑式。

分析：

- `ADD 指令 T5: A + B -> R1`
- `LOAD 指令 T5: MDR -> R1`

所以：

```text
R1in = ADD·T5 + LOAD·T5
```

也可以合并：

```text
R1in = (ADD + LOAD)·T5
```

## 8. 题型四：判断某控制信号是否冲突

### 核心规则

一条总线上同一时刻通常只能有一个部件输出。

所以同一节拍不能同时出现：

```text
PCout 和 MDRout
R1out 和 R2out
ALUout 和 MDRout
```

否则多个部件同时往总线上送数据，会总线冲突。

但同一节拍可以有多个输入信号同时有效，例如：

```text
PCout, MARin, Yin
```

含义是 PC 把数据送上总线，MAR 和 Y 同时接收。

判断口诀：

> 输出只能一个，输入可以多个。

## 9. 题型五：补全指令执行流程

### 常见取指流程

多数题目默认取指流程都差不多：

```text
T0: PC -> MAR
T1: M[MAR] -> MDR, PC + 1 -> PC
T2: MDR -> IR
```

也可能写成：

```text
T0: PC -> MAR
T1: Read
T2: MDR -> IR, PC + 1 -> PC
```

不同教材节拍划分略有差异，但思想相同：

1. PC 给出指令地址。
2. 地址送 MAR。
3. 主存读出指令到 MDR。
4. MDR 送 IR。
5. PC 指向下一条指令。

### 常见执行流程

#### 加法指令 `ADD R1, R2`

含义：

```text
R1 <- R1 + R2
```

可能微操作：

```text
T3: R1 -> A
T4: R2 -> B
T5: A + B -> R1
```

或者单总线结构下：

```text
T3: R1 -> Y
T4: R2 -> bus, ALU加法
T5: Z -> R1
```

#### 访存读数 `LOAD R1, addr`

含义：

```text
R1 <- M[addr]
```

可能微操作：

```text
T3: IR(addr) -> MAR
T4: M[MAR] -> MDR
T5: MDR -> R1
```

#### 存数 `STORE R1, addr`

含义：

```text
M[addr] <- R1
```

可能微操作：

```text
T3: IR(addr) -> MAR
T4: R1 -> MDR
T5: MDR -> M[MAR]
```

#### 无条件转移 `JMP addr`

含义：

```text
PC <- addr
```

可能微操作：

```text
T3: IR(addr) -> PC
```

#### 条件转移 `JZ addr`

含义：如果零标志 `Z=1`，则跳转。

```text
T3: if Z=1 then IR(addr) -> PC
```

对应逻辑式常写为：

```text
PCin = JMP·T3 + JZ·Z·T3
```

## 10. 题型六：画硬布线控制器框图

如果让画框图，按这个结构画：

```text
          操作码
            |
            v
        指令译码器
            |
            v
译码信号 ----\
             \
节拍信号 ------> 组合逻辑电路 ---> 控制信号
             /
状态标志 ----/

时序发生器 ---> 节拍信号
```

文字说明：

> 指令寄存器中的操作码经指令译码器产生指令译码信号；时序发生器产生机器周期和节拍信号；状态条件电路提供标志位和中断请求等条件信号；组合逻辑电路综合这些输入，输出各微操作控制信号。

## 11. 高频易错点

### 1. 硬布线不是“用导线连接一下”

硬布线的本质是：

```text
固定的组合逻辑电路
```

不是临时接线，也不是软件控制。

### 2. 硬布线没有控制存储器

控制存储器、微地址、微指令、微程序是**微程序控制器**的关键词。

硬布线关键词是：

```text
组合逻辑、译码器、节拍、状态条件、速度快、修改困难
```

### 3. 取指阶段通常所有指令都相同

所以取指阶段的控制信号一般不需要乘以 `ADD、LOAD、STORE` 这些指令信号。

例如：

```text
MARin = T0 + LOAD·T3
```

不要写成：

```text
MARin = ADD·T0 + LOAD·T0 + STORE·T0 + LOAD·T3
```

虽然逻辑上可能也对，但考试不简洁。

### 4. 执行阶段要乘以指令译码信号

例如 `LOAD` 的执行阶段读内存：

```text
MemRead = T1 + LOAD·T4
```

不能写成：

```text
MemRead = T1 + T4
```

因为不是所有指令的 `T4` 都读内存。

### 5. 条件转移要乘以状态条件

例如零转移：

```text
PCin = JZ·Z·T3
```

如果写成：

```text
PCin = JZ·T3
```

就变成无条件跳转了。

## 12. 硬布线题目做题总模板

### 简答题

按这个顺序答：

```text
定义 -> 组成 -> 工作原理 -> 优点 -> 缺点 -> 适用场景
```

### 逻辑式题

按这个顺序做：

```text
找控制信号
-> 找哪些微操作用它
-> 写出每个微操作的发生条件
-> 用或连接
-> 能合并就合并
```

### 流程题

按这个顺序写：

```text
先写取指公共流程
再写执行阶段
访存指令一定经过 MAR/MDR
运算指令一定经过寄存器/ALU
转移指令重点改 PC
```

### 判断题

牢记：

```text
硬布线：快、组合逻辑、难改、适合 RISC
微程序：慢一点、控制存储器、易改、适合 CISC
```

## 13. 最后压缩背诵版

硬布线控制器用组合逻辑电路直接产生控制信号。输入包括操作码译码信号、节拍信号和状态条件信号，输出为微操作控制信号。其组成包括指令译码器、时序发生器、状态条件逻辑和组合逻辑电路。优点是速度快，适合 RISC；缺点是结构复杂，设计、调试、修改和扩展困难。

控制信号逻辑式写法：

```text
控制信号 = 所有需要该信号的微操作发生条件相或
```

例如：

```text
MARin = T0 + LOAD·T3
MemRead = T1 + LOAD·T4
PCin = JMP·T3 + JZ·Z·T3
```

最关键区别：

```text
硬布线 = 组合逻辑直接产生控制信号
微程序 = 微指令从控制存储器中读出后产生控制信号
```
