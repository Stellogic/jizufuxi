## 三、必须背的概念

### 1. 冯诺依曼结构

主要思想：

- 数据和指令都用**二进制**表示，指令由**操作码和地址码**组成。
- **程序和数据存放在存储器**中，计算机按程序自动执行，即存储程序、程序控制。
- 指令一般**顺序执行，分支由转移指令实现**。
- 计算机由**运算器、控制器、存储器、输入设备、输出设备**五大部件组成。

冯诺依曼结构：指令和数据放在**同一个存储器**。  
哈佛结构：指令和数据分别放在**两个存储器**。

指令和数据如何区分：

- **时间上：取指周期从内存读出的是指令，执行周期读写的是数据。**
- **空间上：指令流向控制器，数据在内存/寄存器与运算器之间流动。**

### 2. CPU 性能指标

- 处理机字长：运算器一次能完成二进制运算的位数。
- 总线宽度：运算器与存储器等部件之间传输数据的位数。
- 主频：CPU 主时钟频率。
- 时钟周期：主频倒数。
- CPI：平均每条指令需要的时钟周期数。
- MIPS：每秒执行多少百万条指令。

公式：

```text
CPU 执行时间 = CPU 时钟周期数 * 时钟周期
CPU 执行时间 = 指令条数 * CPI / 主频
MIPS = 指令条数 / (程序执行时间 * 10^6)
MIPS = 主频(MHz) / CPI
平均指令周期 = 1 / 指令执行速度
```

### 3. 原码、反码、补码、移码

n 位带符号整数：

- 原码范围：`-(2^(n-1)-1)` 到 `+(2^(n-1)-1)`，有 +0 和 -0。
- 反码范围同原码，也有 +0 和 -0。
- 补码范围：`-2^(n-1)` 到 `2^(n-1)-1`，0 唯一。
- 移码通常用于阶码，真值加偏置；8 位偏置常用 128，IEEE754 单精度阶码偏置是 127。

转换规则：

- 正数：原码、反码、补码数值位相同。
- 负数：原码符号位为 1；反码是原码数值位取反；补码是反码加 1。
- 求负数补码的真值：若最高位为 1，按补码求反加 1 得绝对值，再加负号。
- 8 位补码中特殊值：`1000 0000` 表示 -128，原码和反码不能表示 -128。

补码加减法：

```text
x - y = x + (-y)
```

变形补码溢出判断：

- 双符号位相同：无溢出。
- 双符号位为 `01`：正溢出。
- 双符号位为 `10`：负溢出。

普通补码溢出判断：

- 同号相加，结果异号，溢出。
- 异号相加，不溢出。

### 4. IEEE754 单精度

格式：

```text
S  E(8位)  M(23位)
X = (-1)^S * (1.M) * 2^(E-127)
```

十进制转 IEEE754：

1. 转二进制。
2. 规格化为 `±1.M * 2^e`。
3. 符号位 `S`：正数 0，负数 1。
4. 阶码 `E=e+127`，转 8 位二进制。
5. 尾数 `M` 取小数部分，不写隐藏的 1，补足 23 位。
6. 合并成 32 位，可按 4 位一组转十六进制。

IEEE754 反解：

1. 十六进制转 32 位二进制。
2. 拆出 S、E、M。
3. 算 `e=E-127`。
4. 还原 `(-1)^S * (1.M) * 2^e`。

浮点加减步骤：

0. 零操作数判断
1. 对阶：小阶向大阶看齐，小阶尾数右移。
2. 尾数加减。
3. 规格化：左规或右规，同时调整阶码。
4. 舍入。
5. 判断溢出。

## 四、存储系统做题方法

### 1. 存储容量与芯片扩展

常用公式：

```text
总容量(bit) = 存储单元数 * 字长
芯片数 = 目标总容量(bit) / 单片容量(bit)
地址线根数 = log2(可寻址单元数)
数据线根数 = 字长
```

位扩展：增加字长。  
字扩展：增加存储单元数。  
字位同时扩展：先分组位扩展，再组间字扩展。

例题套路：

1. 先统一单位：K/M/G 和 bit/B。
2. 求目标容量和单片容量。
3. 片数等于二者相除。
4. 数据宽度不够就是位扩展，地址范围不够就是字扩展。
5. 高位地址通常用于片选或译码器选择。

### 2. MAR 和 MDR

- MAR：存放当前 CPU 要访问的内存单元地址，位数由编址单元数决定。
- MDR：存放读出或写入内存的信息，位数通常等于一次读写的数据宽度。

按字节编址：地址数按字节数算，MDR 常按 8 位。  
按字编址：地址数按字数算，MDR 等于字长。

### 3. DRAM 刷新

```text
异步刷新间隔 = 刷新周期 / 行数
集中刷新总时间 = 行数 * 存储周期
死时间率 = 集中刷新总时间 / 刷新周期
```

### 4. 多模块交叉存储器

连续读 m 个字：

```text
顺序方式时间 = mT
交叉方式时间 = T + (m-1)t
带宽 = 传输总位数 / 总时间
```

其中 T 是存储周期，t 是总线传送周期。

## 五、Cache 和虚存

### 1. Cache 性能

```text
命中率 h = Cache 完成存取次数 / 总存取次数
平均访问时间 ta = h * tc + (1-h) * tm
效率 e = tc / ta
```

tc 是 Cache 访问时间，tm 是主存访问时间。

### 2. Cache 地址字段

块内偏移：

```text
块内地址位数 = log2(块大小)
```

直接映射：

```text
Cache 行号位数 = log2(Cache 行数)
标记位数 = 主存地址位数 - 行号位数 - 块内地址位数
Cache 行号 = 主存块号 mod Cache 行数
```

组相联：

```text
组数 = Cache 行数 / 每组行数
组号位数 = log2(组数)
标记位数 = 主存地址位数 - 组号位数 - 块内地址位数
Cache 组号 = 主存块号 mod 组数
```

全相联：

```text
无行号/组号字段
标记位数 = 主存地址位数 - 块内地址位数
```

二维数组命中率套路：

- C 语言数组按行优先存放。
- 若按行访问，连续元素在同一块内：通常 1 次不命中，后面若干次命中。
- 若按列访问，跨行跳跃大，容易映射到同一 Cache 行，命中率可能很低。
- 每块能放多少个数组元素：`块大小 / 元素大小`。

### 3. 虚拟存储器

地址字段：

```text
页内地址位数 = log2(页面大小)
虚页号位数 = 虚拟地址位数 - 页内地址位数
物理页号位数 = 物理地址位数 - 页内地址位数
页表长度 = 2^(虚页号位数)
```

虚实地址转换：

1. 根据页面大小取低位作为页内地址。
2. 高位是虚页号。
3. 查页表得物理页号。
4. 物理地址 = 物理页号 + 页内地址。

缺页替换：

- FIFO：淘汰最早进入主存的页。
- LRU：淘汰最近最久未使用的页。

Cache 缺失和缺页区别：

- Cache 缺失访问主存，代价较小。
- 缺页要访问磁盘，代价很大。

## 六、指令系统与寻址

### 1. 指令格式分析

会算：

```text
操作码位数 = ceil(log2(指令条数))
寄存器字段位数 = log2(寄存器个数)
寻址方式字段位数 = log2(寻址方式种数)
地址字段位数 = 指令字长 - 其他字段位数
```

常见分析语言：

- 单字长/双字长。
- 一地址/二地址/三地址。
- RR 型：两个操作数在寄存器。
- RS 型：一个操作数在寄存器，一个在主存。
- SS 型：两个操作数都在主存。

扩展操作码题：

1. 三地址先占最多地址字段，剩余位给 OP。
2. 没用完的三地址 OP 前缀扩展给二地址。
3. 没用完的二地址 OP 前缀扩展给一地址。
4. 没用完的一地址 OP 前缀扩展给零地址。

### 2. 寻址方式

必须背：

```text
立即寻址：操作数在指令中
直接寻址：EA = A
间接寻址：EA = (A)
寄存器寻址：操作数在寄存器 R 中
寄存器间接寻址：EA = (R)
变址寻址：EA = (X) + A
基址寻址：EA = (B) + A
相对寻址：EA = (PC) + A
```

注意：

- 相对寻址的 PC 通常是取完当前指令后的 PC。
- 按字节编址和按字编址会影响 PC 增量。
- 位移量可能是补码，要能表示正负跳转。

## 七、CPU、微程序、流水线

### 1. 指令周期

取指基本流程：

```text
MAR <- PC
MDR <- M[MAR]
PC <- PC + 指令长度
IR <- MDR
译码
```

取数指令常见流程：

```text
有效地址 -> MAR
MDR <- M[MAR]
目标寄存器 <- MDR
```

存数指令常见流程：

```text
有效地址 -> MAR
源寄存器 -> MDR
M[MAR] <- MDR
```

数据通路题方法：

1. 先找寄存器、ALU、总线、MAR/MDR/IR/PC。
2. 单总线结构一次通常只能有一个输出源。
3. ALU 两个输入若来自同一总线，常需要暂存器 T。
4. 写控制信号通常是 `寄存器in`，读控制信号通常是 `寄存器out`。
5. 微操作按节拍列：每拍写“功能”和“有效控制信号”。

### 2. 微程序控制

概念：

- 微操作：寄存器传送、ALU 运算、读写存储器等最基本操作。
- 微命令：控制一个微操作的控制信号。
- 微指令：一组在一个节拍内可同时发出的微命令，加上下地址/判别字段。
- 微程序：完成一条机器指令的一段微指令序列。
- 控制存储器 CM：存放微程序。

控存容量：

```text
控存容量 = 微指令条数 * 微指令字长
```

微指令字段：

- 微命令字段。
- 判别/测试字段。
- 后继微地址字段。

互斥微命令可用字段译码，能缩短控制字段；可并行的微命令不能放在同一互斥字段里。

### 3. 流水线

公式：

```text
流水线周期 = max(各段时间)
n 个任务、k 段流水线总时间 = (k+n-1) * 流水线周期
吞吐率 = n / 总时间
加速比 = 非流水时间 / 流水时间 = n*k / (k+n-1)
```

五段流水常见：

```text
IF  ID  EX  MEM  WB
```

数据相关：

- RAW：写后读，最常造成阻塞。
- WAR：读后写。
- WAW：写后写。

无转发技术时，后续指令常要等前条指令 WB 后才能读寄存器。若题目说同一寄存器读写不能同周期，则要再多等一拍。

画时空图方法：

1. 第一条从第 1 拍开始依次 IF、ID、EX、MEM、WB。
2. 正常下一条每拍进入。
3. 遇到数据相关，在需要读操作数的 ID 阶段卡住。
4. 后续指令的 IF 也可能因为前面 ID 阻塞而卡住。
5. 最后数完成 WB 的时钟周期。

## 八、总线、磁盘、I/O

### 1. 总线

总线带宽：

```text
总线带宽 = 每个总线周期传输字节数 * 总线时钟频率
```

串行传输：

```text
字符传输率 = 波特率 / 每字符总位数
每字符总位数 = 数据位 + 校验位 + 起始位 + 停止位
```

总线仲裁：

- 集中式仲裁。
- 分布式仲裁。

总线定时：

- 同步定时。
- 异步定时。
- 半同步/分离式定时按教材表述理解。

### 2. 磁盘

```text
磁盘容量 = 每道字节数 * 磁道数 * 记录面数
转速 r/min -> r/60 转/秒
数据传输率 = 每道字节数 * 每秒转数
平均等待时间 = 转一圈时间 / 2 = 60 / 转速 / 2
最高位密度 = 每道字节数 / 最小磁道周长
```

### 3. I/O 方式

CPU 管理外设方式：

- 程序查询：CPU 不断查询外设状态，硬件简单但浪费 CPU。
- 程序中断：外设需要服务时请求中断，CPU 响应后执行中断服务程序。
- DMA：外设和内存直接传送，CPU 只做预处理和后处理。
- 通道：通道执行通道程序，统一管理外设，使 CPU 更少参与。
- 外围处理机：更独立，接近一般处理机。

中断和 DMA 区别：

- 中断方式：请求是为了让 CPU 执行服务程序完成数据传送。
- DMA 方式：数据直接在外设和主存之间传送，中断请求主要用于报告一批数据传送结束，CPU 做后处理。

CPU 响应 I/O 中断条件：

- 开中断。
- I/O 设备有请求且未被屏蔽。
- 经优先级排队后被选中。

响应时间：通常在一条指令执行阶段结束时。

I/O 编址：

- 统一编址：I/O 地址占用主存地址空间，用访存指令访问。
- 独立编址：I/O 地址空间与主存分开，不占主存空间，但需要专用 I/O 指令。

I/O CPU 占比计算：

程序查询：

```text
设备产生一个数据的时间 = 缓冲寄存器字节数 / 数据传输率
每秒查询次数 >= 1 / 上述时间
CPU 占比 = 每秒查询次数 * 每次指令数 * CPI / 主频
```

中断：

```text
每秒中断次数 = 数据传输率 / 每次传输字节数
每次中断周期数 = 中断服务指令数*CPI + 额外开销周期数
CPU 占比 = 每秒中断次数 * 每次中断周期数 / 主频
```

DMA：

```text
每秒 DMA 次数 = 数据传输率 / 数据块大小
CPU 占比 = 每秒 DMA 次数 * 预处理和后处理周期数 / 主频
```

## 九、考前最后背诵清单

### 简答题背诵句

1. 冯诺依曼：二进制、存储程序、程序控制、顺序执行、五大部件。
2. 指令/数据区分：取指周期读出指令，执行周期读写数据；指令流向控制器，数据流向运算器/寄存器/存储器。
3. 软件硬件逻辑等价：同一功能可由软件或硬件实现。
4. Cache：利用程序局部性，在 CPU 和主存之间加入高速小容量存储器，提高平均访问速度。
5. 虚存：用辅存扩展主存逻辑容量，通过地址映射让程序看到更大的地址空间。
6. DMA：DMA 控制器取得总线控制权后，在外设和主存之间直接传送数据。
7. 通道：执行通道指令/通道程序，管理外设并组织数据传送。
8. 微程序：用微指令序列解释执行机器指令。

### 计算题先后顺序

1. 先写单位和地址位数。
2. 再画字段或列公式。
3. 代数计算。
4. 最后写结论和单位。

常见单位坑：

- B 是字节，b 是位。
- 1KB 通常按 `2^10B`。
- 主存按字节编址时，地址位数按字节数算。
- Cache 块大小如果是字节，块内地址按字节算。
- IEEE754 尾数不写隐藏位 1。
- 相对寻址 PC 多数取“取完指令后的 PC”。
- 流水线周期取最长阶段。

## 十、押题式练法

计算题 4 道，建议按这个比例练：

1. 数制/浮点 1 道：原反补移、IEEE754、补码加减溢出、浮点加减。
2. 存储/Cache/虚存 1 道：地址字段、命中率、平均访问时间、页式地址转换。
3. CPU/流水线 1 道：微操作信号、时空图、吞吐率、加速比、阻塞。
4. 总线/I/O/磁盘 1 道：带宽、传输率、CPU 占比、DMA。

应用题 3 道，建议重点练：

1. 指令格式 + 寻址方式求 EA。
2. Cache 映射 + 程序访问命中率。
3. 数据通路/流水线时空图/微操作控制信号。
