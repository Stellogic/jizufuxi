## 三、必须背的概念

### 1. 冯诺依曼结构

主要思想：

- 数据和指令都用**二进制**表示，指令由**操作码和地址码**组成。
- **程序和数据存放在存储器**中，计算机按程序自动执行，即存储程序、程序控制。
- 指令一般**顺序执行，分支由转移指令实现**。
- 计算机由**运算器、控制器、存储器、输入设备、输出设备**五大部件组成。

冯诺依曼结构：指令和数据放在**同一个存储器**。  
哈佛结构：指令和数据分别放在**两个存储器**。

指令和数据如何区分：

- **时间上：取指周期从内存读出的是指令，执行周期读写的是数据。**
- **空间上：指令流向控制器，数据在内存/寄存器与运算器之间流动。**

### 2. CPU 性能指标

- 处理机字长：运算器一次能完成二进制运算的位数。
- 总线宽度：运算器与存储器等部件之间传输数据的位数。
- 主频：CPU 主时钟频率。
- 时钟周期：主频倒数。
- CPI：平均每条指令需要的时钟周期数。
- MIPS：每秒执行多少百万条指令。

公式：

```text
CPU 执行时间 = CPU 时钟周期数 * 时钟周期
CPU 执行时间 = 指令条数 * CPI / 主频
MIPS = 指令条数 / (程序执行时间 * 10^6)
MIPS = 主频(MHz) / CPI
平均指令周期 = 1 / 指令执行速度
```

### 3. 原码、反码、补码、移码

n 位带符号整数：

- 原码范围：`-(2^(n-1)-1)` 到 `+(2^(n-1)-1)`，有 +0 和 -0。
- 反码范围同原码，也有 +0 和 -0。
- 补码范围：`-2^(n-1)` 到 `2^(n-1)-1`，0 唯一。
- 移码通常用于阶码，真值加偏置；8 位偏置常用 128，IEEE754 单精度阶码偏置是 127。

转换规则：

- 正数：原码、反码、补码数值位相同。
- 负数：原码符号位为 1；反码是原码数值位取反；补码是反码加 1。
- 求负数补码的真值：若最高位为 1，按补码求反加 1 得绝对值，再加负号。
- 8 位补码中特殊值：`1000 0000` 表示 -128，原码和反码不能表示 -128。

补码加减法：

```text
x - y = x + (-y)
```

变形补码溢出判断：

- 双符号位相同：无溢出。
- 双符号位为 `01`：正溢出。
- 双符号位为 `10`：负溢出。

普通补码溢出判断：

- 同号相加，结果异号，溢出。
- 异号相加，不溢出。

### 4. IEEE754 单精度

格式：

```text
S  E(8位)  M(23位)
X = (-1)^S * (1.M) * 2^(E-127)
```

十进制转 IEEE754：

1. 转二进制。
2. 规格化为 `±1.M * 2^e`。
3. 符号位 `S`：正数 0，负数 1。
4. 阶码 `E=e+127`，转 8 位二进制。
5. 尾数 `M` 取小数部分，不写隐藏的 1，补足 23 位。
6. 合并成 32 位，可按 4 位一组转十六进制。

IEEE754 反解：

1. 十六进制转 32 位二进制。
2. 拆出 S、E、M。
3. 算 `e=E-127`。
4. 还原 `(-1)^S * (1.M) * 2^e`。

浮点加减步骤：

0. 零操作数判断
1. 对阶：小阶向大阶看齐，小阶尾数右移。
2. 尾数加减。
3. 规格化：左规或右规，同时调整阶码。
4. 舍入。
5. 判断溢出。

## 四、存储系统做题方法

### 1. 存储容量与芯片扩展

常用公式：

```text
总容量(bit) = 存储单元数 * 字长
芯片数 = 目标总容量(bit) / 单片容量(bit)
地址线根数 = log2(可寻址单元数)
数据线根数 = 字长
```

位扩展：增加字长。  
字扩展：增加存储单元数。  
字位同时扩展：先分组位扩展，再组间字扩展。

例题套路：

1. 先统一单位：K/M/G 和 bit/B。
2. 求目标容量和单片容量。
3. 片数等于二者相除。
4. 数据宽度不够就是位扩展，地址范围不够就是字扩展。
5. 高位地址通常用于片选或译码器选择。

### 2. MAR 和 MDR

- MAR：存放当前 CPU 要访问的内存单元**地址**，位数由编址单元数决定。
- MDR：存放读出或写入内存的信息，位数通常等于一次读写的数据宽度。

按字节编址：地址数按字节数算，MDR 常按 8 位。  
按字编址：地址数按字数算，MDR 等于字长。

### 3. DRAM 刷新

```text
异步刷新间隔（如果 DRAM 要在一个刷新周期内把所有行都刷新一遍，那么每隔多长时间刷新一行） = 刷新周期 / 行数
集中刷新总时间（先暂停正常读写，把所有行一次性刷新） = 行数 * 存储周期
死时间率（集中刷新时，存储器不能被正常访问的那段时间） = 集中刷新总时间 / 刷新周期
```

### 4. 多模块交叉存储器

连续读 m 个字：

```text
顺序方式时间 = mT
交叉方式时间 = T + (m-1)t
带宽 = 传输总位数 / 总时间
```

其中 T 是存储周期，t 是总线传送周期。

## 五、Cache 和虚存

### 1. Cache 性能

```text
命中率 h = Cache 完成存取次数 / 总存取次数
平均访问时间 ta = h * tc + (1-h) * tm
效率 e = tc / ta
```

tc 是 Cache 访问时间，tm 是主存访问时间。

### 2. Cache 地址字段

块内偏移：

```text
块内地址位数 = log2(块大小)
```

直接映射：

```text
Cache 行号位数 = log2(Cache 行数)
标记位数 = 主存地址位数 - 行号位数 - 块内地址位数
Cache 行号 = 主存块号 mod Cache 行数
```

组相联：

```text
组数 = Cache 行数 / 每组行数
组号位数 = log2(组数)
标记位数 = 主存地址位数 - 组号位数 - 块内地址位数
Cache 组号 = 主存块号 mod 组数
```

全相联：

```text
无行号/组号字段
标记位数 = 主存地址位数 - 块内地址位数
```

二维数组命中率套路：

- C 语言数组按行优先存放。
- 若按行访问，连续元素在同一块内：通常 1 次不命中，后面若干次命中。
- 若按列访问，跨行跳跃大，容易映射到同一 Cache 行，命中率可能很低。
- 每块能放多少个数组元素：`块大小 / 元素大小`。

### 3. 虚拟存储器

地址字段：

```text
页内地址位数 = log2(页面大小)
虚页号位数 = 虚拟地址位数 - 页内地址位数
物理页号位数 = 物理地址位数 - 页内地址位数
页表长度 = 2^(虚页号位数)
```

虚实地址转换：

1. 根据页面大小取低位作为页内地址。
2. 高位是虚页号。
3. 查页表得物理页号。
4. 物理地址 = 物理页号 + 页内地址。

缺页替换：

- FIFO：淘汰最早进入主存的页。
- LRU：淘汰最近最久未使用的页。

Cache 缺失和缺页区别：

- Cache 缺失访问主存，代价较小。
- 缺页要访问磁盘，代价很大。

## 六、指令系统与寻址

### 1. 指令格式分析

OP + A1 + A2 + A3

op：操作码

A：地址字段

会算：

```text
操作码位数 = ceil(log2(指令条数))（ceil向上取整）
寄存器字段位数 = log2(寄存器个数)
寻址方式字段位数 = log2(寻址方式种数)
地址字段位数 = 指令字长 - 其他字段位数
```

常见分析语言：

- 单字长/双字长。
- 一地址/二地址/三地址。
- RR 型：两个操作数在寄存器。
- RS 型：一个操作数在寄存器，一个在主存。
- SS 型：两个操作数都在主存。

扩展操作码题：扩展的是OP

就是扩展前后的指令都是可以用的。目标是让相同位数的指令尽可能表示更多的操作

1. 三地址先占最多地址字段，剩余位给 OP。
2. 没用完的三地址 OP 前缀扩展给二地址。
3. 没用完的二地址 OP 前缀扩展给一地址。
4. 没用完的一地址 OP 前缀扩展给零地址。

### 2. 寻址方式

必须背：

```text
立即寻址：操作数在指令中
直接寻址：EA = A
间接寻址：EA = (A)
寄存器寻址：操作数在寄存器 R 中
寄存器间接寻址：EA = (R)
变址寻址：EA = (X) + A
基址寻址：EA = (B) + A
相对寻址：EA = (PC) + A
```

注意：

- 相对寻址的 PC 通常是取完当前指令后的 PC。
- 按字节编址和按字编址会影响 PC 增量。
- 位移量可能是补码，要能表示正负跳转。

## 七、CPU、微程序、流水线

### 1. 指令周期

取指基本流程：

```text
MAR <- PC。PC （程序计数器，存折下一条指令地址）的值送到 MAR（内存地址寄存器。告诉内容要访问哪个地址）
MDR <- M[MAR]（取出MAR内容，放到MDR，mdr内存数据寄存器用来暂存从内存读出的数据）
PC <- PC + 指令长度（pc自动加上当前指令长度，指向下一条指令）
IR <- MDR（把刚从内存取出的指令，从 MDR 送到 IR，指令寄存器，专门存放当前正在执行的指令。）
译码
```

取数指令常见流程：

**MAR 放地址，MDR 放数据。**

取数是“内存 -> CPU”，存数是“CPU -> 内存”。

```text
有效地址 -> MAR
MDR <- M[MAR]
目标寄存器 <- MDR
```

存数指令常见流程：

```text
有效地址 -> MAR
源寄存器 -> MDR
M[MAR] <- MDR
```

数据通路题方法：

PC：存下一条指令地址

IR：存当前指令

MAR：存访存地址

MDR：存访存数据

通用寄存器：如 R0、R1、R2

ALU：做加减与或等运算

1. 先找寄存器、ALU、总线、MAR/MDR/IR/PC。
2. 单总线结构一次通常只能有一个输出源。
3. ALU 两个输入若来自同一总线，常需要暂存器 T。
4. 写控制信号通常是 `寄存器in`，读控制信号通常是 `寄存器out`。
5. 微操作按节拍列：每拍写“功能”和“有效控制信号”。

### 2. 微程序控制

概念：

- 微操作：寄存器传送、ALU 运算、读写存储器等最基本操作。
- 微命令：控制一个微操作的控制信号。
- 微指令：一组在一个节拍内可同时发出的微命令，加上下地址/判别字段。
- 微程序：完成一条机器指令的一段微指令序列。
- 控制存储器 CM：存放微程序。

控存容量：

```text
控存容量 = 微指令条数 * 微指令字长
```

微指令字段：

- 微命令字段。
- 判别/测试字段。
- 后继微地址字段。

互斥微命令可用字段译码，能缩短控制字段；可并行的微命令不能放在同一互斥字段里。



### 2.5 硬布线/硬连线控制器

硬布线控制器也叫硬连线控制器，核心思想是：用组合逻辑电路根据指令操作码、节拍信号、状态条件和标志位，直接产生各个控制信号。

必背对比：

- 硬布线控制器：控制信号由逻辑门电路直接生成，速度快；结构固定，设计和修改困难，适合 RISC 或指令系统较简单的机器。
- 微程序控制器：控制信号由控制存储器中的微指令产生，设计规整，修改和扩展方便；需要读取微指令，速度相对慢，适合 CISC 或指令系统复杂的机器。

硬布线控制信号来源：

```text
控制信号 = f(指令操作码, 时序节拍, 状态条件, 标志位)
```

简答模板：

硬布线控制器不把控制过程写成微程序，而是把每条机器指令在各节拍应发出的微命令固化为逻辑电路。CPU 取出并译码指令后，控制器根据操作码、节拍脉冲和状态标志，经组合逻辑产生控制信号，控制数据通路完成指令执行。

选择题易混：

- 速度：硬布线通常快于微程序。
- 灵活性：微程序强于硬布线。
- 修改指令系统：微程序可改控存内容，硬布线通常要改电路。
- 复杂指令系统：更适合微程序控制。

### 3. 流水线

公式：

```text
流水线周期 = max(各段时间)
n 个任务、k 段流水线总时间 = (k+n-1) * 流水线周期
吞吐率 = n / 总时间
加速比 = 非流水时间 / 流水时间 = n*k / (k+n-1)
```

五段流水常见：

```text
IF（取指令）  ID  EX  MEM  WB
```

```
IF  = Instruction Fetch，取指
从内存中取出下一条要执行的指令。

ID  = Instruction Decode，译码 / 读寄存器
分析这条指令要干什么，同时读取需要的寄存器操作数。

EX  = Execute，执行
真正进行运算，比如加减、比较、地址计算等。

MEM = Memory Access，访存
如果是 load/store 指令，就在这里读写内存。
普通运算指令这一阶段可能只是经过一下。

WB  = Write Back，写回
把运算结果或从内存读出的数据写回寄存器。

```

```txt
周期:  1   2   3   4   5   6   7
I1:   IF  ID  EX  MEM WB
I2:       IF  ID  EX  MEM WB
I3:           IF  ID  EX  MEM WB
```

数据相关：后一条指令和前一条指令使用了同一个寄存器，而且读写顺序会影响结果

- **RAW：写后读，最常造成阻塞。**
- WAR：读后写。
- WAW：写后写。

raw阻塞

```
I1: ADD R1, R2, R3
I2: SUB R4, R1, R5
```

```
周期:  1   2   3   4   5   6
I1:   IF  ID  EX  MEM WB
I2:       IF  ID  EX  MEM WB
```

I2 在 ID 阶段要读 R1。
但 R1 要等 I1 的 WB 阶段才写回。

所以 I2 不能在第 3 拍读寄存器，必须卡在 ID 阶段等。

如果允许同周期先写后读，那么 I2 可以在第 5 拍 ID 读到 I1 WB 写回的结果：

```
周期:  1   2   3   4   5   6   7   8
I1:   IF  ID  EX  MEM WB
I2:       IF  ID  ID  ID  EX  MEM WB
```

无转发技术时，后续指令常要等前条指令 WB 后才能读寄存器。若题目说同一寄存器读写不能同周期，则要再多等一拍。

```
周期:  1   2   3   4   5   6   7   8   9
I1:   IF  ID  EX  MEM WB
I2:       IF  ID  ID  ID  ID  EX  MEM WB
```

后续指令为什么也会被卡住

画时空图方法：

1. 第一条从第 1 拍开始依次 IF、ID、EX、MEM、WB。
2. 正常下一条每拍进入。
3. 遇到数据相关，在需要读操作数的 ID 阶段卡住。
4. 后续指令的 IF 也可能因为前面 ID 阻塞而卡住。
5. 最后数完成 WB 的时钟周期。

## 八、总线、磁盘、I/O

### 1. 总线

总线带宽：

```text
总线带宽 = 每个总线周期传输字节数 * 总线时钟频率
```

串行传输：

```text
波特率 ：每秒传几个码元，码元是最小单位一般就是比特bit
字符传输率 = 波特率 / 每字符总位数
每字符总位数 = 数据位 + 校验位 + 起始位 + 停止位
```

总线仲裁：当多个主设备都想使用总线时，到底谁先用？

- 集中式仲裁。由一个统一的仲裁器来决定谁获得总线使用权。
- 分布式仲裁。没有集中的仲裁器，各个设备自己通过某种规则协商，决定谁获得总线。

总线定时：总线传输过程中，各个操作什么时候开始、什么时候结束，如何协调发送方和接收方。

- 同步定时。所有总线操作都由统一时钟控制。
- 异步定时。没有统一时钟，靠“握手信号”协调传输。“你准备好了吗？我准备好了，那开始”。
- 半同步/。平时大家按时钟节拍走，如果有慢设备，就允许它喊“等等”。
- 分离式定时。普通传输中，主设备发出请求后可能要一直占着总线等结果。分两个阶段请求阶段：主设备发送地址和命令。响应阶段：从设备准备好数据后，再申请总线返回结果。中间等待期间，总线可以让其他设备使用。

### 2. 磁盘

磁道就是一个一个圆环

靠近圆心的磁道 = 内道

靠近盘片边缘的磁道 = 外道

```text
磁盘容量 = 每面磁道数 × 记录面数 × 每道字节数。
磁盘容量 = 记录面数 × 每面磁道数 × 每道扇区数 × 每扇区字节数
转速 r/min -> r/60 转/秒
数据传输率（磁盘在连续读/写时，每秒能传多少数据。转一圈只读一个磁道） = 每道字节数 * 每秒转数
平均等待时间（磁头已经移动到目标磁道了，但目标扇区还没转到磁头下面，于是还要等一会儿。可能刚好在，0；可能恰好一圈，认为随机分布） = 转一圈时间 / 2 = 60 / 转速 / 2
最高位密度 = 每道字节数 / 最小磁道周长
```

### 3. I/O 方式

CPU 管理外设方式：

- 程序查询：CPU 不断查询外设状态，硬件简单但浪费 CPU。
- 程序中断：外设需要服务时请求中断，CPU 响应后执行中断服务程序。
- DMA：外设和内存直接传送，CPU 只做预处理和后处理。
- 通道：通道执行通道程序，统一管理外设，使 CPU 更少参与。
- 外围处理机：更独立，接近一般处理机。

中断和 DMA 区别：

- 中断方式：请求是为了让 CPU 执行服务程序完成数据传送。
- DMA 方式：数据直接在外设和主存之间传送，中断请求主要用于报告一批数据传送结束，CPU 做后处理。

CPU 响应 I/O 中断条件：

- 开中断。
- I/O 设备有请求且未被屏蔽。
- 经优先级排队后被选中。

响应时间：通常在一条指令执行阶段结束时。

I/O 编址：

- 统一编址：I/O 地址占用主存地址空间，用访存指令访问。
- 独立编址：I/O 地址空间与主存分开，不占主存空间，但需要专用 I/O 指令。

I/O CPU 占比计算：

程序查询：

```text
设备产生一个数据的时间 = 缓冲寄存器字节数 / 数据传输率
每秒查询次数 >= 1 / 上述时间
CPU 占比 = 每秒查询次数 * 每次指令数 * CPI / 主频
```

中断：

```text
每秒中断次数 = 数据传输率（单位时间传了多少数据） / 每次传输字节数
每次中断周期数 = 中断服务指令数*CPI + 额外开销周期数
CPU 占比 = 每秒中断次数 * 每次中断周期数 / 主频
```

DMA：

```text
每秒 DMA 次数 = 数据传输率 / 数据块大小
CPU 占比 = 每秒 DMA 次数 * 预处理和后处理周期数 / 主频
```

## 九、考前最后背诵清单

### 简答题背诵句

1. 冯诺依曼：二进制、存储程序、程序控制、顺序执行、五大部件。
2. 指令/数据区分：取指周期读出指令，执行周期读写数据；指令流向控制器，数据流向运算器/寄存器/存储器。
3. 软件硬件逻辑等价：同一功能可由软件或硬件实现。
4. Cache：利用程序局部性，在 CPU 和主存之间加入高速小容量存储器，提高平均访问速度。
5. 虚存：用辅存扩展主存逻辑容量，通过地址映射让程序看到更大的地址空间。
6. DMA：DMA 控制器取得总线控制权后，在外设和主存之间直接传送数据。
7. 通道：执行通道指令/通道程序，管理外设并组织数据传送。
8. 微程序：用微指令序列解释执行机器指令。
9. 硬布线控制：由组合逻辑电路直接产生控制信号，速度快但修改困难。

### 计算题先后顺序

1. 先写单位和地址位数。
2. 再画字段或列公式。
3. 代数计算。
4. 最后写结论和单位。

常见单位坑：

- B 是字节，b 是位。
- 1KB 通常按 `2^10B`。
- 主存按字节编址时，地址位数按字节数算。
- Cache 块大小如果是字节，块内地址按字节算。
- IEEE754 尾数不写隐藏位 1。
- 相对寻址 PC 多数取“取完指令后的 PC”。
- 流水线周期取最长阶段。

## 十一、二次核查补漏

这一节是对照标黄课件、两份习题解答和本地复习笔记后补上的漏项。前面章节已经覆盖主干，但以下内容容易在单选、简答或应用题里被单独拎出来。

### 1. 字符与校验码

- ASCII 通常是 7 位编码，可表示 128 个字符。
- 汉字编码可记：GB2312、GBK、Unicode、UTF-8、UTF-16。选择题常问“字符集”和“编码方式”区别，Unicode 是统一字符集，UTF-8/UTF-16 是编码实现。
- 奇偶校验：增加 1 位校验位，使 1 的个数满足奇数或偶数。能发现奇数个比特错误，尤其单比特错误；不能定位错误，也不能纠错，偶数个错误可能漏检。

### 2. SRAM、DRAM、ROM

- SRAM：速度快，不需刷新，常用于 Cache；集成度低、成本高。
- DRAM：容量大、成本低，需要刷新，常用于主存。
- ROM：掉电不丢失，主要用于读；常见分类按教材记 ROM、PROM、EPROM、EEPROM/Flash。
- DRAM 刷新方式：集中刷新、分散刷新、异步刷新。计算题重点仍是刷新间隔、集中刷新总时间、死时间率。

### 3. Cache 写策略

命中时：

- 写直达/全写法：写 Cache 的同时写主存。优点是主存总是新数据，缺点是写慢。
- 写回法：先只写 Cache，替换时再写回主存。优点是减少主存写次数，缺点是要脏位，控制复杂。

未命中时：

- 写分配：先把块调入 Cache，再写。
- 非写分配：不调入 Cache，直接写主存。

### 4. 页式、段式、段页式比较

- 页式：页大小固定，地址为页号 + 页内地址。页不一定连续。优点是无外部碎片、管理方便；缺点是有内部碎片，页表可能很大。
- 段式：地址为段号 + 段内地址，段表含段基址和段长。段要连续。优点是符合程序逻辑，便于共享和保护；缺点是有外部碎片。
- 段页式：先分段、段内分页，地址为段号 + 页号 + 页内地址。兼有逻辑性和固定分页管理，但地址转换更复杂。

### 5. 原码/补码乘除与 ALU 进位

原码阵列乘法：

1. 符号位单独处理，结果符号为两个符号位异或。
2. 数值位按无符号乘法计算。
3. 拼上符号位得到原码乘积。

补码阵列乘法：

1. 先看操作数符号，负数需要算前求补得到绝对值。
2. 数值位按无符号乘法。
3. 若乘积应为负，算后再求补。

加减交替除法：

- 先把被除数、除数按题意变成定点小数。
- 余数为正时减除数，余数为负时加除数。
- 每一步左移、判余数符号、确定商位。

ALU 先行进位常用式：

```text
Gi = AiBi
Pi = Ai + Bi
Ci = Gi + Pi*C(i-1)
C1 = G1 + P1C0
C2 = G2 + P2G1 + P2P1C0
C3 = G3 + P3G2 + P3P2G1 + P3P2P1C0
C4 = G4 + P4G3 + P4P3G2 + P4P3P2G1 + P4P3P2P1C0
```

### 6. 微指令编码与相斥/相容微操作

- 水平型微指令：并行性强、速度快，但字长长。
- 垂直型微指令：编码短、节省控存，但译码多、速度较慢。
- 直接控制法：一个微命令对应一位，快但费位。
- 字段译码法：把互斥微命令放进同一字段，节省位数，但同字段内不能并行。
- 字段位数：若一组有 n 个互斥微命令，还要留“无操作”编码，则 `2^k >= n + 1`。
- 控存有 `2^k` 个单元时，后继微地址字段至少 k 位。
- 相斥微操作：同一时刻不能同时发生，例如同一总线多个部件同时输出。
- 相容微操作：可在同一节拍并行发生，例如不争用同一资源的寄存器打入和运算控制。


### 6.5 硬布线控制器补充

- 硬布线控制器靠组合逻辑直接产生微命令，不需要控制存储器。
- 输入通常包括：指令译码结果、节拍/时序信号、条件码或状态标志、中断/异常等条件。
- 优点：速度快，控制信号产生直接。
- 缺点：电路复杂，设计调试困难，指令系统改变时不易修改。
- 和微程序控制器比较时，按“速度 vs 灵活性”答：硬布线速度快，微程序灵活、易扩展。

### 7. CISC 与 RISC

- CISC：指令多、格式复杂、寻址方式多、执行时间不固定，控制器复杂。
- RISC：指令少、格式规整、寻址方式少，多数指令周期短，Load/Store 结构，寄存器多，适合流水线，更多依赖编译器优化。

### 8. 总线与串行通信

- 总线按信息分：地址总线、数据总线、控制总线。
- 总线按位置分：片内总线、系统总线、外部总线。
- 总线仲裁：集中式、分布式。
- 总线定时：同步、异步、半同步/分离式。
- 异步串行一帧通常是：起始位 + 数据位 + 校验位 + 停止位。字符传输率一定要用波特率除以“一帧总位数”。

### 9. 显示设备

显示类题目重点不多，但单选/小计算可能出现：

```text
显存容量 = 水平像素数 * 垂直像素数 * 每像素位数
显示带宽 = 显存容量 * 刷新率
```

概念：分辨率、像素、颜色深度、刷新率、显存容量。

### 10. 中断与 DMA 细节

中断过程：

```text
中断请求 -> 中断判优 -> 中断响应 -> 保存现场 -> 执行中断服务程序 -> 恢复现场 -> 中断返回
```

中断方式与 DMA 区别补充：

- 中断是程序切换，需要保护和恢复现场；DMA 除预处理和后处理外不占用 CPU。
- 中断传送靠 CPU 执行程序，DMA 传送靠硬件。
- DMA 请求优先级通常高于普通中断请求。
- 中断能处理异常事件，DMA 主要用于成组数据传送。

DMA 传送方式常见三类：

- 停止 CPU 访问主存/成组传送：DMA 连续占用总线，速度快但 CPU 暂停明显。
- 周期挪用/周期窃取：DMA 每次占用一个或几个存储周期，CPU 被短暂停顿。
- 透明传送：DMA 利用 CPU 不访问主存的空闲周期传送，不影响 CPU，但速度较慢。

### 11. 题型核查结论

没有发现新的大题型。仍按老师给的四类准备：单选 15 道、简答 4 道、计算 4 道、应用 3 道。

但应用题要再细分成 5 种小形态：

1. 存储器扩展框图。
2. 指令格式/寻址方式/扩展操作码。
3. Cache/虚存地址字段和程序访问命中率。
4. CPU 数据通路、指令周期、微操作控制信号。
5. 流水线时空图、数据相关、阻塞原因。

### 12. 低频但可能偷袭的点

- 存储器技术指标：存储容量、存取时间、存储周期、存储器带宽、位价、可靠性。存取时间是一次访问从启动到完成的时间，存储周期是两次连续访问之间的最短间隔。
- 双端口存储器：同一个存储器有两套相互独立的地址、数据和控制端口，可支持两个部件同时访问；若同时访问同一单元，需要冲突检测和仲裁。
- 堆栈寻址：操作数默认在栈顶，由栈指针 SP 指出，常用于子程序调用、中断现场保护、表达式求值。
- 程序查询方式基本过程：CPU 启动外设 -> 读状态寄存器 -> 判断 Ready 位 -> 未就绪继续查 -> 就绪后传送数据 -> 判断是否传完。
- 无条件传送/程序直接控制：默认外设总是准备好，不查询状态，硬件简单但适用范围窄。
- 中断方式涉及的问题：中断源识别、中断判优、中断屏蔽、中断嵌套、现场保护和恢复、中断向量。
- 通道类型：字节多路通道适合多台低速设备交叉传送；数组多路通道适合多台高速设备成组交叉传送；选择通道一次只为一台高速设备服务。
- 不同外设接入主机要解决的问题：速度差异、数据格式转换、命令传送、状态反馈、数据地址识别和控制。
- 性能指标易混：吞吐率是单位时间完成多少任务，响应时间是完成一个任务花多久。
